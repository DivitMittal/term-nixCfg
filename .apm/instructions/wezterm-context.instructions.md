## Module Layout

`wezterm.lua` is the entry point. It calls each sibling module as a function:

```
wezterm.lua → options(wezterm, config)
            → binds(wezterm, config)
            → smartSplits(wezterm, config)
            → tabline(wezterm, config)
```

Every module returns a single `function(wezterm, config)` that mutates `config` in place. No module stores its own state or returns values.

## Key Bindings (binds.lua)

Leader key: `CTRL+r` (800 ms timeout). Escape hatch: `LEADER+CTRL+r` sends a literal `Ctrl+r` to the terminal (useful for shell history search). Splits: `LEADER+s` (vertical), `LEADER+v` (horizontal). Copy mode: `LEADER+Enter`.

## Plugins

- **smart-splits.nvim** (`smartSplits.lua`): `CTRL+Arrow` moves focus between panes/Neovim splits; `ALT+Arrow` resizes panes.
- **tabline.wez** (`tabline.lua`): Tab bar pinned to the bottom, hidden when only one tab exists. Theme is hard-coded to `"Dracula"` — independent of the `cyberpunk` window color scheme.

## Color Scheme Dependency

`options.lua` sets `config.color_scheme = "cyberpunk"`. This TOML file is **generated by home-manager** (in `config/home/wezterm.nix`) from `OS-nixCfg/lib/palette.nix` and written to `$XDG_CONFIG_HOME/wezterm/colors/cyberpunk.toml`. It does not exist in this repo. Do not add it.

`config.window_background_opacity = 0.85` must stay in sync with `opacity.terminal` in `OS-nixCfg/lib/palette.nix`.

## Adding a New Module

1. Create `emulators/wezterm/newmodule.lua` exporting `function(wezterm, config)`.
2. Add `require "newmodule"(wezterm, config)` in `wezterm.lua`.
3. Add `"newmodule.lua"` to the `weztermFiles` list in `config/home/wezterm.nix` — without this the file won't be symlinked into XDG config.
